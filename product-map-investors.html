The Problem

Ground crews at regional airports — the people who clean, service, and turn around aircraft between flights — coordinate using group texts, printed checklists, and verbal handoffs. There is no shared system. Tasks get missed. Supplies run out without warning. Shift changes leave the next crew uninformed. And when staff turns over every 6 to 12 months, institutional knowledge walks out the door with them.

The Market
✈ Regional Aviation
Hundreds of regional airports across North America, each with ground crews managing aircraft turns, crew cars, and supply chains. No dominant software player serves this segment.
🏨 Hospitality
Hotel housekeeping teams face identical problems — task checklists, room turnover tracking, supply management, shift handoffs. The core product translates directly.
🏗 Facilities Management
Facilities teams managing buildings, vehicles, and maintenance schedules share the same coordination gaps. Same platform, different skin.
🛫 FBOs & Ground Handlers
Fixed-base operators and commercial ground handling companies manage large ramp crews across multiple gates. High-value, high-volume target segment.
The Insight
Why now, why this
  • The tools these teams use today — paper, WhatsApp, spreadsheets — are not being replaced by enterprise software because enterprise software is too expensive, too complex, and not built for shift workers on a ramp
  • A mobile-first, zero-friction app that installs like a website and costs $50–$200/month per location sits exactly in the gap between "nothing" and "enterprise"
  • Skycare was built from the inside — by someone who works this operation — which means the product is grounded in real daily use, not assumptions
  • It is already live. It already works. The proof of concept is not a prototype — it is a production system used every day at CYKF
What It Is

Skycare is a mobile-first operations platform for ground crews. It replaces paper checklists, group texts, and verbal handoffs with a single app every team member runs on their phone — no installation, no accounts, no IT department required. It works offline, sends push notifications, and tracks everything from daily tasks to live aircraft positions.

Stack
Frontend
  • Vanilla HTML / CSS / JavaScript — no framework, no build pipeline
  • Progressive Web App — installable, offline-capable
  • Service worker — network-first caching, push receiver
  • Leaflet 1.9.4 — interactive fleet map
  • 4 UI themes — Premium and Standard × Night and Day
Backend
  • Firebase Firestore — all application data, real-time sync
  • Firebase Storage — audit photos
  • Node.js on GitHub Actions — all notification delivery
  • web-push + VAPID — cross-platform push notifications
  • OpenSky Network API — live ADS-B aircraft position data
  • cron-job.org — scheduled workflow triggers every 5 minutes

The Margin Story

The platform runs at $0/month today on free infrastructure tiers. Adding a paying client requires creating a new Firebase project and deploying a config file — no new servers, no new engineers, no marginal cost. Revenue from client one is almost entirely profit. This scales until Firebase paid tiers kick in, well beyond the first 20–30 clients.

What $100/month × 50 locations looks like
